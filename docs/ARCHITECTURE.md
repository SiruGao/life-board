# Architecture

## 当前架构

Socratic Kernel 当前是零依赖静态 PWA：

- `index.html`：应用壳与导航
- `styles.css`：响应式视觉系统
- `app.js`：状态、规则引擎、审议流程与本地数据管理
- `manifest.webmanifest`：可安装 Web App 元数据
- `sw.js`：离线缓存
- `icon.svg`：应用图标

用户数据使用浏览器 `LocalStorage` 保存。应用没有后端、账户系统、分析 SDK 或第三方数据传输。

## 核心模块

### 1. Intake

强制用户在系统追问之前写下问题、初步立场、依据和确信度。

### 2. Pattern Detector

使用可检查的本地规则发现绝对化表达、外部评价牵引、紧迫性放大、判断外包和流畅性信任等线索。输出是待核验提示，不是诊断。

### 3. Inquiry Composer

根据问题类型、挑战强度和已发现线索，从概念、证据、反方、价值、后果、责任与实验等维度编排问题。

### 4. Commitment Gate

只有在用户逐题作答后，才进入最终判断页面。最终结论、行动和责任必须由用户填写。

### 5. Memory Ledger

保存审议过程、信心变化和重复线索。数据可导出、导入、单条删除和全部删除。

## 下一阶段目标架构

后续增加服务端能力时，应保持三层隔离：

1. **本地自主权层**：决定是否需要 AI、保存私密偏好、控制删除与导出。
2. **审议编排层**：分类请求属于信息、执行、分析还是价值判断，并选择追问协议。
3. **模型与知识层**：调用可替换模型、哲学原典检索与反迎合审查器。

任何大模型输出都必须经过：来源标注、不确定性声明、反迎合检查和用户最终确认。
