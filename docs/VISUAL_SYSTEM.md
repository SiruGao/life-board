# Socratic Kernel 视觉系统

## 核心气质

Socratic Kernel 的界面不是传统 AI 聊天工具，也不是通用 SaaS 仪表盘。它的视觉定位是：

> 哲学编辑部 × 当代艺术馆 × 安静奢华

页面应让用户感受到停顿、审视和责任，而不是效率焦虑、娱乐刺激或模型能力炫耀。

## 设计原则

### 1. 留白不是空缺

留白用于区分：

- 用户自己的判断；
- 系统提出的问题；
- 事实与解释；
- 可以行动的部分与暂时未知的部分。

页面不通过堆叠卡片填满空间。

### 2. 排版承担叙事

中文标题使用系统可用的宋体或古典衬线字体栈，信息和操作使用克制的无衬线字体。大标题不是装饰，而是让核心判断在视觉上具有重量。

### 3. 减少“组件感”

避免大量圆角卡片、渐变按钮、气泡和胶囊标签。界面主要使用：

- 细线网格；
- 编辑编号；
- 非对称分栏；
- 大字号与小字号的强对比；
- 方形边界；
- 展览目录式导航。

### 4. 动效表达思考，而非刺激

轨道和核心图形使用缓慢、低频运动，象征一个判断被多个问题反复环绕。交互动效不应制造倒计时、奖励或错失恐惧。

所有非必要动画必须在 `prefers-reduced-motion: reduce` 下关闭。

### 5. 艺术性不能破坏可用性

- 文本对比度保持清晰；
- 键盘焦点可见；
- 360px 宽度下不横向溢出；
- 主要按钮不依赖颜色单独表达状态；
- 本地、隐私和 AI 状态必须可理解；
- 不加载外部字体和图库，保持离线能力。

## 色彩

| Token | 色值 | 用途 |
| --- | --- | --- |
| Carbon | `#0c0c0b` | 首屏、主按钮、最强视觉背景 |
| Ink | `#12110f` | 正文、边线、主要信息 |
| Bone Paper | `#eee8de` | 页面主背景 |
| Light Paper | `#f8f4ed` | 内容面板与文本反差 |
| Oxblood | `#5b1820` | 关键动作、强调、AI/价值冲突 |
| Dark Oxblood | `#351014` | 深色侧栏与隐私区 |
| Aged Copper | `#b18a55` | 编号、轨道节点、低频点缀 |

酒红只用于真正需要强调的判断节点，不能成为大面积营销色。

## 排版

### Display / 标题

```css
"Iowan Old Style", "Baskerville", "Times New Roman",
"Noto Serif SC", "Songti SC", STSong, serif
```

### Interface / 信息与操作

```css
Inter, ui-sans-serif, -apple-system, BlinkMacSystemFont,
"Segoe UI", "PingFang SC", "Microsoft YaHei", sans-serif
```

不从外部 CDN 加载字体，以保障隐私、速度和离线使用。

## 首页结构

```text
展览目录式导航
↓
黑色主视觉：品牌宣言 + 轨道核心雕塑
↓
首次使用引导（仅新用户）
↓
产品宣言：工具可以替你加速，不能替你承担
↓
五种审议方式目录
↓
练习信号
↓
最近审议
```

首页的第一目标不是解释全部功能，而是在数秒内建立品牌识别与产品立场。

## 组件语言

### 按钮

- 方形或近方形；
- 主按钮使用墨黑或酒红；
- 次按钮使用细线边框；
- hover 只做轻微位移和颜色反转；
- 不使用弹跳、发光或持续呼吸。

### 表单

- 以底部细线代替封闭输入框；
- 长文本使用衬线字体，提高“书写”而不是“填写表格”的感受；
- 模式选择使用编辑网格和实心/空心圆状态；
- 标签使用小字号大字距。

### 卡片

- 通过网格边线形成整体，不让每张卡片像独立浮层；
- hover 时整块反色；
- 标题承担主要视觉重量；
- 元数据退到二级层。

## 品牌图形

核心图形由三部分组成：

1. 多条不同倾角的轨道：代表定义、证据、反方、价值和后果；
2. 酒红色核心：代表用户当前的判断；
3. 铜金节点：代表一个能够改变判断的新事实或问题。

图形不代表“AI 大脑”，避免常见神经网络、机器人和聊天气泡符号。
