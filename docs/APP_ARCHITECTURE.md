# 从网页版到原生应用

Socratic Kernel 不需要重写三套产品。推荐采用共享核心、分阶段封装的路线。

## 产品形态

### 第一阶段：Web App / PWA

当前版本已经具备：

- 响应式网页界面；
- PWA 安装；
- 离线本地规则引擎；
- 本地档案；
- 可选 AI 模型网关；
- Vercel Serverless Functions。

这是最快验证产品价值、使用流程和订阅意愿的版本。

### 第二阶段：桌面应用

使用 Tauri 封装现有前端，目标平台：

- macOS；
- Windows；
- Linux。

桌面端新增价值：

- 全局快捷键启动审议；
- 系统托盘；
- 本地加密数据库；
- 从其他应用读取用户主动复制或分享的文本；
- 与浏览器扩展联动；
- 可选本地模型运行。

### 第三阶段：移动应用

使用 Capacitor 封装共享前端，目标平台：

- iOS；
- Android。

移动端新增价值：

- 系统分享菜单：把网页、文章或选中文字发送到“阅读质疑”；
- 本地通知：在用户主动设置的时间重新检验判断；
- 生物识别解锁私密档案；
- 安全存储认证令牌；
- App Store 和 Google Play 分发。

## 共享架构

```text
packages/core
  审议数据结构、规则引擎、提示协议、导入导出

apps/web
  浏览器和 PWA 界面

apps/desktop
  Tauri 壳与桌面系统能力

apps/mobile
  Capacitor 壳与 iOS / Android 系统能力

api
  统一模型网关、认证、额度、订阅与同步
```

当前仓库仍是单体静态结构。完成模型网关和浏览器扩展验证后，再迁移为上述 monorepo，避免过早增加工程复杂度。

## 统一模型网关

所有客户端都调用同一个 Socratic Kernel API，而不是分别内置供应商密钥：

```text
Web ─────────┐
Desktop ─────┼── Socratic Kernel Gateway ── 多家模型供应商
Mobile ──────┤
Extension ───┘
```

网关统一承担：

- 用户认证；
- 订阅和用量；
- 模型路由；
- API Key 管理；
- 请求长度和速率限制；
- 反迎合审查；
- 供应商故障回退；
- 用户同意和隐私策略。

## 数据层演进

### 当前

`LocalStorage`，适合无登录 MVP。

### 桌面和移动 Beta

本地 SQLite 或平台安全存储，档案默认仅在设备上。

### 订阅版

端到端加密同步：

- 服务端只保存密文；
- 加密密钥由用户设备持有；
- 用户可关闭同步并删除云端密文；
- 管理员不能读取个人审议正文。

## 推荐开发顺序

1. 完成 Web AI 网关；
2. 完成浏览器扩展 MVP；
3. 验证 30–50 名真实用户的完成率和留存；
4. 抽取共享核心包；
5. 开发 Tauri 桌面 Beta；
6. 开发 Capacitor 移动 Beta；
7. 增加账号、订阅、额度和加密同步；
8. 准备 App Store / Google Play 合规材料。

## 不建议现在做的事

- 同时维护独立 Web、iOS、Android 三套业务代码；
- 在安装包中保存供应商 API Key；
- 在没有留存数据前开发复杂支付和团队后台；
- 默认上传完整思考档案；
- 用通知、签到或连续天数制造依赖。
